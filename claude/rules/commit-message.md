## 基本ルール（git.md の補完）

- 説明部分は日本語・1行・50文字以内
- 変更内容が一目でわかる形にする
- 「fix」「更新」だけの曖昧なメッセージは不可
  - NG: `fix`
  - OK: `docs: pre-commitルールにyamllint迂回手順を追加`

## プレフィックス（必要に応じて）

コミットテンプレート `~/.gitmassage`（リポジトリの `gitmassage`）と対応させる。
`<プレフィックス>: <日本語の説明>` の形式で書く。

| プレフィックス | 用途                                                             |
| -------------- | ---------------------------------------------------------------- |
| `feat:`        | 新機能                                                           |
| `fix:`         | バグ修正                                                         |
| `docs:`        | ドキュメントのみの変更                                           |
| `style:`       | フォーマットの変更（スペース・セミコロンなど、動作に影響しない） |
| `refactor:`    | リファクタリング（機能追加・バグ修正を含まない）                 |
| `perf:`        | パフォーマンス向上                                               |
| `test:`        | 不足テストの追加・既存テストの修正                               |
| `ci:`          | CIのための変更                                                   |
| `chore:`       | その他の変更（ソース・テストの変更を含まない）                   |
| `revert:`      | 変更を元に戻す                                                   |
| `WIP:`         | 作業中の変更                                                     |
