## 基本ルール（git.md の補完）

- 日本語・1行・50文字以内
- 変更内容が一目でわかる形にする
- 「修正」「更新」だけの曖昧なメッセージは不可
  - NG: 「修正」
  - OK: 「pre-commitルールにyamllint迂回手順を追加」

## プレフィックス（必要に応じて）

| プレフィックス | 用途                 | 例                                |
| -------------- | -------------------- | --------------------------------- |
| `追加:`        | 新規ファイル・機能   | `追加: securityルール`            |
| `修正:`        | バグ修正・既存変更   | `修正: git.mdのブランチ操作`      |
| `削除:`        | ファイル・設定削除   | `削除: 不要な古い設定`            |
| `更新:`        | 依存関係・バージョン | `更新: settings.jsonのプラグイン` |

## 禁止

- AI署名（🤖 / Co-Authored-By）は git.md の通り付与しない
- 空コミットは作らない
