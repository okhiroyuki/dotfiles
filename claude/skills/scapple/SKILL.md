# Scapple言語化スキル

`.scap`ファイル（ブレインストーミングツールScappleの保存形式）を読み込み、ノード同士の関係を理解した上で、曖昧な点をユーザーに質問しながら文章化する。

## 前提条件

`scapple`コマンドがセットアップ済みであること（セットアップ手順は`tools/scapple/README.md`を参照）。

## 手順

1. **CLIでパースする**（トークンを消費しない前処理）

   ```bash
   scapple <file.scap> --format json
   ```

   生のXMLをそのまま読み込まないこと。座標・色・フォントなどの言語化に不要な情報はCLIが既に除去済みなので、出力されたJSON（`notes: [{id, text, connectedTo, pointsTo}]`、`stacks: number[][]`）だけを読む。

2. **構造を把握する**
   - `pointsTo` は有向の矢印（因果・依存など）、`connectedTo` は無向の線（関連・並列など）
   - `stacks` は縦積みでグループ化されたノート群。配列の並び順どおりの一連の流れ（手順・段階）として扱う
   - 孤立ノード（`connectedTo`・`pointsTo`・どの`stacks`にも属さず、他ノードからも参照されていない）や、関係の意味が文面から読み取れない接続を洗い出す
   - `text` が `[image: ファイル名]` の形式のノートは、テキストではなく画像が貼られていたことを示すプレースホルダ。画像の中身はCLIから渡されないため、内容を憶測せず何が描かれているかをユーザーに確認する

3. **不明点をユーザーに質問する**
   - 一度に3つ以内にまとめる
   - 意図・関係性にフォーカスし、技術的になりすぎない
   - ユーザーの回答を踏まえて、必要なら追加で質問する

4. **言語化する**
   - ユーザーが形式を指定していなければ、自然な文章（prose）で出力する
   - `markdown` / `outline` / `json` を指定された場合はその形式に従う

## 成功基準

- CLIの生JSON出力をそのままユーザーに見せず、必ず言語化した結果を返している
- 接続関係（有向・無向の別）を無視せず、文章に反映している
- 曖昧な接続・孤立ノードを質問せずに憶測で埋めていない
