## rcm (rcup/lsrc) との整合性

- ファイル・ディレクトリを追加/削除したら、`rcrc` の `EXCLUDES` に該当のエントリが必要か（あるいは不要になった記載が残っていないか）を確認する
- 変更後は `lsrc -d .` を実行し、シンボリックリンク対象が意図通りになっているか確認する

## rcmコマンド一覧

ユーザーがコマンドを忘れた場合、以下を参照して説明する（詳細: [rcm(7) tutorial](http://thoughtbot.github.io/rcm/rcm.7.html)）。

| コマンド | 用途                                                                                                                                |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `rcup`   | dotfilesを実際のホームディレクトリにシンボリックリンクとしてインストール/更新する（本リポジトリでは `rcup -B private` / `-B work`） |
| `rcdn`   | `rcup` の逆。シンボリックリンクを解除する                                                                                           |
| `mkrc`   | 既存のdotfileをdotfilesディレクトリに取り込む（tag指定可）                                                                          |
| `lsrc`   | 管理対象のdotfileと、それぞれのリンク先を一覧表示する（ドライラン確認用）                                                           |
| `rcrc`   | 上記コマンドの設定ファイル（本リポジトリでは [rcrc](../../rcrc)）                                                                   |

- host-private / host-work は rcm の `host-<HOSTNAME>` 規約を利用しており、`rcup -B private` / `-B work` でタグ名をホスト名として扱わせている
